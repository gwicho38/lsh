# LSH v3.1.1 Release Notes

**Release Date:** 2025-11-29

## Changes from v3.1.0

### Extended Global Mode Support

The `-g/--global` flag is now available on all top-level commands:

**Newly Added:**
- `lsh init -g` - Initialize global workspace at `$HOME`
- `lsh doctor -g` - Health check for global workspace
- `lsh create -g` - Create `.env` file in `$HOME`
- `lsh clear -g` - Clear metadata (already operates globally)

**Previously Available (v3.1.0):**
- `lsh push -g`
- `lsh pull -g`
- `lsh sync -g`
- `lsh list -g`
- `lsh get -g`
- `lsh set -g`
- `lsh status -g`
- `lsh info -g`
- `lsh env -g`
- `lsh delete -g`

### Command Details

**`lsh init -g`**
- Creates configuration in `~/.env` instead of `./.env`
- Skips `.gitignore` updates (since `$HOME` is not a git repo)
- Shows global workspace indicator during setup

**`lsh doctor -g`**
- Checks health of global workspace configuration
- Skips git repository checks in global mode
- Shows global workspace location in output

**`lsh create -g`**
- Creates new `.env` file in `$HOME`
- Supports `--template` flag for common variables
